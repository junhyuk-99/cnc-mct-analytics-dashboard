# 재사용 모듈 및 확장 후보

이 문서는 공개 저장소를 확장할 때 재사용할 수 있는 모듈과, 원본 프로덕션 시스템에서 **개념만** 가져와 추가로 재구성할 수 있는 후보를 정리합니다.

원본 시스템의 파일 경로·클래스명·내부 구조는 이 문서에 포함하지 않습니다. 재구성 원칙은 항상 동일합니다: **소스 복사가 아니라 중립 네이밍 + 합성 데이터 계약으로 재작성**하고, 커밋 전 시크릿 스캔을 통과해야 합니다. (상세 기준: [SECURITY.md](SECURITY.md))

## 1. 이 저장소에서 바로 재사용 가능한 모듈

| 모듈 | 위치 | 재사용 포인트 |
| --- | --- | --- |
| 델타 보정 규칙 | `backend/.../rollup/DeltaRule.java` | 누적 카운터 → 델타 변환이 필요한 모든 신호 유형에 그대로 적용 가능 (순수 함수, 단위 테스트 포함) |
| 롤업 파이프라인 | `backend/.../rollup/` | 스케줄러/서비스/리포지토리/인덱스/백필 구조는 신호 종류를 늘려도 동일 패턴으로 확장 |
| 읽기 API 패턴 | `backend/.../controller`, `.../service` | `ApiResponse` 래퍼 + 날짜 범위 파서 + 요약 컬렉션 조회 패턴 |
| 합성 데이터 생성기 | `scripts/generate_sample_data.py` | 새 컬렉션 추가 시 생성 함수만 추가하면 시드 파이프라인 재사용 |
| 프론트 차트/필터 | `frontend/src/components/` | KPI 카드, 차트, 날짜/설비 필터 컴포넌트 |

## 2. 추가 재구성 후보 (원본 시스템의 개념)

원본 프로덕션 시스템은 이 공개 저장소보다 넓은 분석 모듈(절삭 일/월/년, 설비 이력, 성능 기간 분석, 예지 알람, 스핀들/팬, 공구 사용량, 마스터 관리 등)을 포함했습니다. 공개용으로 재구성할 가치가 있는 후보:

| 후보 | 개념 | 재구성 시 유의점 |
| --- | --- | --- |
| 월간/연간 절삭 대시보드 | 시간 버킷을 일 → 월 → 년으로 상향 집계 | 롤업 타깃 컬렉션을 상위 기간으로 2차 집계 |
| 설비 이력 뷰 | 설비별 상태·이벤트 타임라인 | `status_history` 기반, 페이지네이션 필요 |
| 예지 알람 | 임계값 기반 사전 경고 | 임계 규칙을 합성 데이터에 맞게 재정의 |
| 성능 기간 비교 | 기간 A/B 지표 비교 | 기존 요약 컬렉션 쿼리 재사용 |
| 설비 마스터 관리 | 설비/캘린더 마스터 CRUD | 공개 저장소의 읽기 전용 원칙 유지 여부 검토 |

## 3. 공개 저장소에 가져오지 않는 것

다음 범주는 어떤 형태로도 이 저장소에 재구성·이식하지 않습니다:

- 인증/인가, 사용자·비밀번호 처리, 계정 복구 흐름
- 운영 DB 스키마 전체, 실제 컬렉션·필드명, 실제 설비 식별자
- 파일 업로드/다운로드, 인쇄/익스포트 등 업무 문서 흐름
- 운영 배포 스크립트, 인프라 구성, 내부 네트워크 정보
- 비공개 저장소의 Git 히스토리
